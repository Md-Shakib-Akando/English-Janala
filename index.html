Frequently Asked Questions

1. The difference between var, let, and const?
var: Function-scoped, can be redeclared, hoisted with undefined. let: Block-scoped, cannot be redeclared, hoisted without initialization. const: Block-scoped, cannot be reassigned, must be initialized.
2. The difference between map(), forEach(), and filter()?
map(): Returns a new array with transformed elements. forEach(): Iterates over elements but does not return a new array. filter(): Returns a new array with elements that pass a condition.
3. Explain arrow functions and how they are different from regular functions?
Arrow Functions (=>):Shorter syntax.No this binding (inherits from parent scope).Cannot be used as constructors. Regular Functions (function):Has its own this.Can be used as a constructor.Can have arguments object.
4. How JavaScript Promises work?
A Promise represents an asynchronous operation with 3 states:Pending – Initial stateFulfilled – Successfully completed (.then()).Rejected – Failed (.catch()).
5. How closures work in JavaScript?
A closure is a function that remembers its lexical scope even when executed outside of it.Use Cases: Data privacy, function factories, event handlers.
